🧱 How to build it inside OrahDEX (simple version)

1. Create an internal microservice

Name: orahdex-email-service

2. Use Postal as the engine

Postal gives you:

• API
• SMTP
• logs
• bounce handling


3. Connect it to your existing Brevo SMTP (optional)

If your email already works well, you can keep Brevo as the delivery layer.

4. Add an Email tab inside OrahDEX admin

Features:

• Logs
• Templates
• Delivery status
• Bounce reports


5. Add an internal API

Example:

POST /internal/email/send
{
  "to": "user@example.com",
  "template": "withdrawal_alert",
  "data": { "amount": "0.5 ETH" }
}


6. Add webhooks

So OrahDEX knows:

• delivered
• bounced
• opened
