✅ 1. A DEX only shows tokens that exist in its Token Registry

Every DEX has a canonical list of tokens it supports.

Your DEX already has this structure:

/api/tokens


or internally:

const TOKENS = {
  ETH: { symbol: "ETH", decimals: 18, chainId: 1, address: null },
  USDT: { symbol: "USDT", decimals: 6, chainId: 1, address: "0xdAC17F..." },
  USDC: { symbol: "USDC", decimals: 6, chainId: 1, address: "0xA0b869..." },
  ...
}


If a token is not in this registry, the DEX cannot:

• show its balance
• show its price
• show it in the swap list
• show it in the portfolio
• show it in liquidity pools


So to “print” USDT/USDC on your DEX, you add them to the registry.

---

✅ 2. The DEX reads balances using the registry

Your backend already has:

GET /api/balances/:wallet


This endpoint loops through the registry and fetches:

• native balance (ETH)
• ERC‑20 balances (USDT, USDC, etc.)


Example:

for (const token of TOKENS) {
  if (token.address === null) {
    // native ETH
    balance = await provider.getBalance(wallet);
  } else {
    // ERC20
    balance = await erc20(token.address).balanceOf(wallet);
  }
}


This is how USDT/USDC appear in the Portfolio screen.
