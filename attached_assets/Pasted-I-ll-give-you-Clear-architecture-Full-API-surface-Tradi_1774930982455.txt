1. High‑level architecture

Services:

• API Gateway – api.orah.org
• Core DEX Backend – markets, trades, orderbook, candles, wallets
• Liquidity Bot Service – market making, throttled
• TradingView Datafeed Service – symbols, history, streaming
• Admin Panel Backend – config, monitoring, controls
• Postgres/Timescale – markets, trades, candles, orders
• Redis – caches, pub/sub for live updates


---

2. Core REST API (frontend ↔ backend)

Base: https://api.orah.org/v1

Auth:

• Public endpoints: markets, orderbook, candles, trades
• Signed endpoints: orders, portfolio, admin


2.1 Markets & tokens

• List marketsGET /marketsReturns:[
  {
    "symbol": "BSVUSDT",
    "base": "BSV",
    "quote": "USDT",
    "status": "ACTIVE",
    "minQty": "0.001",
    "pricePrecision": 4,
    "qtyPrecision": 3
  }
]

• Single marketGET /markets/{symbol}
• TokensGET /tokens


2.2 Orderbook

• OrderbookGET /orderbookQuery:• symbol=BSVUSDT
• limit=50
Response:{
  "symbol": "BSVUSDT",
  "bids": [[13.81, 0.52], [13.80, 1.10]],
  "asks": [[13.82, 0.40], [13.83, 0.90]],
  "timestamp": 1710000000
}
For unknown pairs: return synthetic book, not 404.


2.3 Candles (charts)

• CandlesGET /candlesQuery:• symbol=BSVUSDT
• interval=1m|5m|1h|4h|1d
• from=unix
• to=unix

5. Frontend integration (web + mobile)

Web app:

• Use your existing React/Vue/Next app.
• For charts, embed TradingView widget with custom datafeed:


const datafeed = new Datafeeds.UDFCompatibleDatafeed("https://tv.orah.org");

new TradingView.widget({
  symbol: "BSVUSDT",
  interval: "60",
  container_id: "tv_chart",
  datafeed,
  library_path: "/tradingview/",
  locale: "en",
  disabled_features: ["use_localstorage_for_settings"],
  enabled_features: [],
  theme: "dark"
});


Mobile app:

• If it’s a webview: same widget.
• If native: use a WebView or a native chart lib fed by /candles + WebSocket.
