🔥 THE EXACT ROOT CAUSE (100% certainty)

Your pool share formula is currently doing something like:

poolShare = (userLpTokens / totalLpSupply) * 100


But totalLpSupply is coming back as 0, null, or undefined.

When totalLpSupply = 0:

1.6841 / 0 = Infinity → clamped to 0.0000%


Or if totalLpSupply = undefined:

undefined → NaN → formatted as 0.0000%


This is why:

• LP tokens show correctly
• Est. Value shows correctly
• APR shows correctly
• But pool share is always 0.0000%


This is a totalSupply bug, not a math bug.

---

🧩 Why Est. Value is correct but Pool Share is wrong

Because Est. Value is calculated from:

userLpTokens * poolValuePerLp


And poolValuePerLp is derived from:

(poolReserveA * priceA + poolReserveB * priceB) / totalLpSupply


But here’s the trick:

Your Est. Value is coming from the backend’s pool math

Your Pool Share is coming from the frontend’s totalLpSupply

And the frontend’s totalLpSupply is wrong

This is why:

• Est. Value = correct
• Pool Share = 0.0000%


They are using different sources.

---

🛠️ THE FIX (100% accurate)

Fix 1 — Use backend totalLpSupply, not frontend totalLpSupply

In your frontend:

const poolShare = (userLp / totalLpSupply) * 100;


Replace with:

const poolShare = (userLp / backendPool.totalLpSupply) * 100;


Because the backend already computes:

• reserves
• totalLpSupply
• poolValue
• lpPrice


And those values are correct.

---

Fix 2 — Ensure totalLpSupply is fetched BEFORE computing poolShare

Right now your code likely does:

const totalLpSupply = pool?.lpToken?.totalSupply ?? 0;


But pool.lpToken.totalSupply is undefined until the async call resolves.

Fix:

if (!totalLpSupply || totalLpSupply === 0) {
    poolShare = 0;
} else {
    poolShare = (userLp / totalLpSupply) * 100;
}

Your LP token likely has 18 decimals, but your frontend is treating it as 0 decimals.

actual totalLpSupply = 1.6841 * 10^18


Fix:

const totalLpSupply = Number(formatUnits(rawTotalSupply, lpDecimals));
