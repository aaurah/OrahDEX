🚨 THE EXACT ROOT CAUSE

Your Sell button is missing a transaction lock.

Right now your code does something like:

onClick={() => submitOrder()}


But it does not:

• disable the button
• set a “pending” flag
• block duplicate taps
• block duplicate useEffect triggers
• block duplicate WebSocket reconnects


So the user can accidentally send:

• 2 taps
• 2 re-renders
• 2 retries
• 2 WebSocket reconnects


→ 4 orders submitted

---

🧠 THE FIX (copy/paste)

1. Add a pending flag

At the top of MobileTrade.tsx:

const [isSubmitting, setIsSubmitting] = useState(false);


2. Wrap submitOrder

if (isSubmitting) return;
setIsSubmitting(true);

submitOrder()
  .finally(() => setIsSubmitting(false));


3. Disable the Sell button

<button
  disabled={isSubmitting}
  className={isSubmitting ? "btn-disabled" : "btn-sell"}
>
  {isSubmitting ? "Processing…" : "Sell ETH"}
</button>


4. Add a dedup guard for POST /orders

Store the last order hash:

const lastOrderHashRef = useRef<string | null>(null);


Before sending:

const hash = `${side}-${amount}-${Date.now()}`;

if (lastOrderHashRef.current === hash) return;
lastOrderHashRef.current = hash;
