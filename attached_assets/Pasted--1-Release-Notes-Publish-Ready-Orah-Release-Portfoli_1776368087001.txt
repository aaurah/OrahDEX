 1. Release Notes — Publish‑Ready

Orah Release — Portfolio, Markets, and AI Stability Update

Overview

This release delivers a full stabilization of the Markets, Portfolio, and AI subsystems.
All data flows now derive directly from chain + DB, eliminating drift and ensuring consistent, real‑time correctness across the entire platform.

---

Key Improvements

1. Portfolio Overhaul

• Expanded ERC‑20 registry across all supported chains (15–35 tokens per chain)
• Switched pricing source from /api/dex/prices → /api/markets?quote=USDT
• Accurate AUD totals for all supported tokens
• Correct 24h change and price display
• Mobile + desktop parity via shared hooks


2. Markets Engine Upgrade

• Full dynamic category generation from DB
• 953 markets now visible across all quote‑asset tabs
• No static lists, no drift, no missing pairs
• Real‑time BSV block height and market status


3. AI Route Hardening

• Added AbortSignal.timeout(25_000) to all AI calls
• Added res.headersSent guards to prevent late writes
• Reduced max_completion_tokens from 8192 → 256–512
• Eliminated “Cannot set headers after they are sent” crashes
• Server remains stable under AI load


4. Server Stability

• Zero unhandled rejections
• Zero Express header errors
• Zero auto‑restarts during verification
• All workflows (Markets, Portfolio, AI) running clean


---

Impact

• Accurate balances
• Accurate pricing
• Complete market visibility
• Stable AI endpoints
• Fully deterministic API behavior
• Production‑ready stability
