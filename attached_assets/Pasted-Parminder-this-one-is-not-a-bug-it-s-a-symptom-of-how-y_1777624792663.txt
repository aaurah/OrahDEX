Parminder — this one is not a bug, it’s a symptom of how your current backend + frontend are working together.

When you refresh the page and the pairs list increases or decreases, it always comes from one of these four causes — and in your case, it’s almost certainly #1 and #2 combined.

Let me break it down cleanly.

---

⭐ 1. Your pairs list is built dynamically from the database

Right now OrahDEX does not have a static list of markets.

Instead, your Markets page loads pairs from:

marketsTable


which is populated by:

• real orders
• bot orders
• synthetic orders
• liquidity bot
• swap activity
• internal price engine
• sometimes LetsExchange metadata (if you imported it)


So when you refresh:

✔ If new bot orders were inserted → new pairs appear

✔ If old bot orders expired → pairs disappear

✔ If synthetic pairs were generated → list grows

✔ If synthetic pairs were cleaned → list shrinks

This is why the list is unstable.

---

⭐ 2. Your liquidity bot is constantly creating & deleting orders

Your liquidity bot:

• places orders
• cancels orders
• replaces orders
• rotates pairs
• updates depth


Every time it does this, the set of active markets changes.

So when you refresh:

✔ If the bot touched a pair → it appears

✔ If the bot removed a pair → it disappears

This is normal for a bot‑driven orderbook.

---

⭐ 3. Synthetic pairs are being generated on demand

You previously added logic:

“If pair has no data, generate synthetic candles and synthetic orderbook.”

This means:

✔ When the frontend requests a pair → backend creates synthetic data

✔ When synthetic data expires → pair disappears

So the list fluctuates.

---

⭐ 4. Your Markets page only shows pairs with recent activity

Your code filters markets by:

• volume
• trades
• orderbook depth
• price engine activity


If a pair has no recent activity, it gets dropped.

If a pair suddenly gets:

• a bot order
• a synthetic candle
• a swap
• a price update


…it reappears.

---

⭐ So the REAL reason your pairs increase/decrease on refresh:

Because your markets list is not static — it is generated from live orderbook + bot activity + synthetic data.

Every refresh = new snapshot of a constantly changing system.

---

⭐ How to FIX this (3 clean options)

Option A — Static markets list (recommended)

Create a table:

markets
  - base
  - quote
  - enabled
  - type (spot, synthetic, external)


Then show only these pairs, regardless of orderbook activity.

This is how Binance, Bybit, OKX do it.

---

Option B — Hybrid list

Show:

• Static pairs (always visible)
• Dynamic pairs (only if active)


This gives stability + flexibility.

---

Option C — LetsExchange import

Import all 5,700+ pairs from LetsExchange and mark them:

type = "external"


Then show:

• internal pairs (orderbook)
• external pairs (LE)


This gives you a stable global list.

---

✔ Use Option A for internal markets

✔ Use Option C for external markets

✔ Merge them into one stable Markets page

This stops the flickering, the randomness, and the “pairs disappearing” problem forever.
