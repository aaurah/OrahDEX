1. Wallet: build a signed OrderIntent

From the wallet, everything is one object:

type OrderKind = 'SPOT' | 'FUTURES';
type OrderType = 'MARKET' | 'LIMIT';

type OrderIntent = {
  kind: OrderKind;          // SPOT or FUTURES
  type: OrderType;          // MARKET or LIMIT
  pair: string;             // e.g. "BSV/USDT"
  side: 'BUY' | 'SELL';
  price?: string;           // required for LIMIT
  amount: string;
  expiry: number;           // unix seconds
  nonce: string;            // random, unique
  fundingRef: string;       // "ledger:..." or "margin:..." or UTXO id
  walletAddress: string;
  signature: string;
};


Wallet flow:

export function buildOrderIntent(params, wallet) {
  const nonce = crypto.randomUUID();
  const expiry = Math.floor(Date.now() / 1000) + 60 * 5; // 5 min

  const fundingRef = selectAndLockFunds(params, wallet); 
  // SPOT → spot bucket / UTXO
  // FUTURES → margin bucket

  const payload = {
    ...params,
    nonce,
    expiry,
    fundingRef,
    walletAddress: wallet.address,
  };

  const signature = wallet.sign(canonicalIntentPayload(payload));

  return { ...payload, signature };
}

2. API: verify + lock funding via a single fundingVerifier

On the server, never inline this logic in routes again — always go through one place:

export async function verifyAndLockFunding(intent: OrderIntent, db) {
  verifySignature(intent);          // walletAddress + signature

  if (intent.kind === 'SPOT') {
    return lockSpotFunds(intent, db);      // touches user_balances
  }

  if (intent.kind === 'FUTURES') {
    return lockFuturesMargin(intent, db);  // touches futures_margin_accounts
  }

  throw new Error('Unsupported order kind');
}


In orders.ts:

const intent = validateOrderIntent(req.body);
const funding = await verifyAndLockFunding(intent, db);

const order = await db.insert(ordersTable).values({
  // ...
  fundingRef: intent.fundingRef,
  nonce: intent.nonce,
  expiry: intent.expiry,
});

4. The invariants that mean “it’s fixed”

If these are true, your system is logically sound:

• Every order row has: funding_ref, nonce, expiry.
• No order is accepted unless verifyAndLockFunding succeeds.
• Spot settlement only touches user_balances.
• Futures settlement only touches futures_margin_accounts.
• Wallet always shows three numbers per asset:• free spot
• locked spot
• futures margin (available + locked)
